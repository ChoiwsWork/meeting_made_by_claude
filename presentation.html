⚡ 기술적 결정사항

🎭 Fake DataSource 전략

백엔드 없이 개발 가능하도록 FakeConferenceDataSource, FakeUserDataSource를 인터페이스 뒤에 배치. MutableStateFlow로 실시간 업데이트 시뮬레이션. 향후 실제 구현체로 교체만 하면 됨.

🔄 Reactive State Management

DataSource → Repository → UseCase → ViewModel까지 Flow 체인으로 연결. ViewModel에서 stateIn(WhileSubscribed(5_000))으로 라이프사이클 인식 수집. UI는 collectAsStateWithLifecycle() 사용.

🏷️ Type-safe Navigation

kotlinx.serialization으로 @Serializable destination 클래스 정의. 문자열 라우트 대신 컴파일 타임에 타입 안전성 보장. 네비게이션 인수를 SavedStateHandle.toRoute()로 역직렬화.

📋 UI 모델 분리

목록 화면용 경량 ConferenceSummary를 별도 정의해 도메인 모델과 UI 관심사 분리. 도메인 변경이 목록 UI에 미치는 영향 최소화.

💉 Hilt DI 전략

Repository · DataSource를 @Singleton으로 바인딩. 모듈별 @Module로 바인딩 분리 (ConferenceRepositoryModule, UserRepositoryModule). Hilt 집계는 :app 클로저를 통해 전이적으로 동작.

🎬 Compose Preview

@PreviewLightDark로 라이트/다크 상태 동시 미리보기. 결정론적 테스트 데이터(고정 타임스탬프)와 dynamic color 비활성화로 일관된 Preview 보장.

🔧 컨벤션 플러그인 (build-logic)
